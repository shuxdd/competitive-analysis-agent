# 智能竞品分析Agent - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目名称
智能竞品分析Agent (Competitive Analysis Agent)

### 1.2 项目背景
企业市场竞争情报收集依赖人工搜索，效率低、覆盖不全、更新滞后。产品经理、运营人员和市场分析师需要花费大量时间（通常2-3天）手动收集和整理竞品信息，且容易遗漏关键变化。

### 1.3 项目目标
构建一个端到端的竞品分析Agent，能够：
- 自动采集竞品的官网信息、社交媒体动态、产品更新、用户评价等多源数据
- 经过结构化分析后输出可视化对比报告
- 帮助产品和运营团队快速掌握竞争格局
- 将竞品分析报告产出时间从人工2天降至5分钟

### 1.4 项目价值
| 维度 | 人工方式 | Agent方式 | 提升 |
|------|---------|----------|------|
| 时间成本 | 2-3天/次 | 5分钟/次 | 99%+ |
| 数据覆盖 | 3-5个源 | 5+个源 | 60%+ |
| 更新频率 | 月度 | 实时/按需 | 持续 |
| 分析一致性 | 因人而异 | 标准化 | 100% |

- 痛点：信息分散、更新不及时、分析不够深入

### 3.2 核心功能详细设计

#### F1: 竞品档案自动建库
**功能描述**：输入竞品名称，自动采集并建立竞品档案

**输入**：
- 竞品名称（必填）
- 竞品官网URL（可选）
- 行业领域（可选）

**处理流程**：
1. 解析输入，生成搜索策略
2. 调用搜索工具获取基础信息
3. 爬取官网获取详细信息
4. LLM提取结构化数据
5. 存入知识库

**输出**：
```json
{
  "competitor_id": "comp_001",
  "name": "竞品名称",
  "basic_info": {
    "company_name": "公司全称",
    "founded": "成立时间",
    "location": "所在地",
    "funding": "融资情况",
    "employees": "员工规模"
  },
  "products": [
    {
      "name": "产品名称",
      "description": "产品描述",
      "features": ["功能1", "功能2"],
      "pricing": {
        "model": "定价模式",
        "tiers": [
          {"name": "基础版", "price": "¥99/月"},
          {"name": "专业版", "price": "¥299/月"}
        ]
      }
    }
  ],
  "target_market": "目标市场",
  "key_differentiators": ["差异化点1", "差异化点2"]
}
```

**验收标准**：
- [ ] 输入竞品名称后30秒内完成基础信息采集
- [ ] 结构化数据准确率≥85%
- [ ] 支持中英文竞品

#### F2: 多源数据采集
**功能描述**：从多个数据源采集竞品信息

**数据源矩阵**：
| 数据源 | 采集内容 | 采集方式 | 更新频率 |
|--------|---------|---------|---------|
| 官网 | 产品功能、定价、公司信息 | Selenium爬虫 | 按需/定时 |
| App Store/Google Play | 用户评价、评分、版本更新 | API/爬虫 | 每日 |
| 社交媒体 | 动态、营销活动、用户反馈 | API/爬虫 | 每日 |
| 行业报告 | 市场份额、行业趋势 | 搜索+下载 | 月度 |
| 新闻资讯 | 公司动态、融资、合作 | SerpAPI | 每日 |

**采集策略**：
- 增量采集：只采集新增/变更内容
- 反爬处理：随机UA、请求间隔、代理池
- 容错机制：单源失败不影响其他源采集
- 数据清洗：去重、格式化、质量校验

**验收标准**：
- [ ] 支持5+数据源
- [ ] 单次采集成功率≥90%
- [ ] 增量采集效率提升50%+

#### F3: 结构化对比分析
**功能描述**：基于采集数据进行多维度对比分析

**分析维度**：

**3.3.1 功能矩阵对比**
```markdown
| 功能模块 | 我方产品 | 竞品A | 竞品B | 竞品C |
|---------|---------|-------|-------|-------|
| 用户管理 | ✅ | ✅ | ✅ | ✅ |
| 数据分析 | ✅ | ✅ | ❌ | ✅ |
| API接口 | ✅ | ❌ | ✅ | ✅ |
| 移动端 | ❌ | ✅ | ✅ | ❌ |
```

**3.3.2 SWOT分析（自动生成）**
```markdown
## 竞品A SWOT分析

### 优势 (Strengths)
- 品牌知名度高，用户基数大
- 产品功能完整，覆盖主要场景

### 劣势 (Weaknesses)
- 定价较高，中小企业难接受
- 移动端体验一般

### 机会 (Opportunities)
- 市场增长迅速，新进入者少
- 技术迭代快，有差异化空间

### 威胁 (Threats)
- 新竞品不断涌现
- 用户需求变化快
```

**3.3.3 价格策略对比**
```markdown
| 产品 | 免费版 | 基础版 | 专业版 | 企业版 |
|------|--------|--------|--------|--------|
| 我方 | ✅ | ¥99/月 | ¥299/月 | 定制 |
| 竞品A | ❌ | ¥199/月 | ¥499/月 | 定制 |
| 竞品B | ✅ | ¥79/月 | ¥199/月 | ¥999/月 |
```

**验收标准**：
- [ ] 功能矩阵对比准确率≥85%
- [ ] SWOT分析逻辑合理，无明显错误
- [ ] 价格数据实时性≤24小时

#### F4: 智能报告生成
**功能描述**：基于分析结果自动生成结构化报告

**报告类型**：
| 报告类型 | 内容 | 生成时间 | 适用场景 |
|---------|------|---------|---------|
| 快速概览 | 竞品基本信息+核心差异 | 1分钟 | 日常了解 |
| 标准分析 | 完整功能对比+SWOT+定价 | 3分钟 | 产品规划 |
| 深度研究 | 全面分析+趋势预测+建议 | 5分钟 | 战略决策 |

**报告结构（标准分析）**：
```markdown
# 竞品分析报告：[竞品名称]
> 生成时间：2024-01-15 | 分析范围：[行业]

## 1. 执行摘要
[500字概述主要发现]

## 2. 竞品概览
### 2.1 公司背景
### 2.2 产品矩阵
### 2.3 市场定位

## 3. 功能对比分析
### 3.1 核心功能矩阵
### 3.2 功能差异分析
### 3.3 功能评分对比

## 4. 定价策略分析
### 4.1 定价模式对比
### 4.2 价格竞争力评估

## 5. SWOT分析
### 5.1 竞品A SWOT
### 5.2 竞品B SWOT
### 5.3 综合对比

## 6. 用户口碑分析
### 6.1 好评关键词
### 6.2 差评关键词
### 6.3 情感倾向

## 7. 趋势与建议
### 7.1 行业趋势
### 7.2 差异化机会
### 7.3 行动建议

## 附录
- 数据来源
- 采集时间
- 详细数据表
```

**输出格式**：
- Markdown（默认）
- PDF（可导出）
- JSON（API接口）

**验收标准**：
- [ ] 报告结构完整，包含所有必要章节
- [ ] 数据引用准确，可追溯
- [ ] 生成时间符合预期（快速<1分钟，标准<3分钟，深度<5分钟）

#### F5: 智能问答（RAG）
**功能描述**：基于已采集数据进行自然语言问答

**问答场景**：
- "竞品A和竞品B的核心差异是什么？"
- "最近一个月竞品有什么重大更新？"
- "竞品的定价策略有什么变化？"
- "用户对竞品的主要不满是什么？"

**技术实现**：
- 基于Chroma向量数据库的RAG检索
- 支持多轮对话，保持上下文
- 回答附带数据来源引用

**验收标准**：
- [ ] 问答准确率≥80%
- [ ] 响应时间<5秒
- [ ] 支持追问和上下文理解

## 6. 数据流程

### 6.1 核心数据流
```
用户输入
    ↓
[任务解析] → 确定分析类型、竞品列表、分析维度
    ↓
[数据采集] → 多源并行采集
    ├── Web Search → SerpAPI
    ├── Web Scraping → Selenium
    ├── App Store → API
    └── Social Media → API
    ↓
[数据处理] → 清洗、去重、格式化
    ↓
[信息抽取] → LLM结构化抽取
    ↓
[知识存储] → Chroma向量数据库 + 结构化JSON
    ↓
[分析引擎] → 多维度对比分析
    ↓
[报告生成] → LLM + 模板
    ↓
输出：结构化报告 / 问答回答 / API响应
```

### 6.2 数据模型
```
Competitor（竞品）
├── basic_info: 公司基本信息
├── products: 产品列表
├── updates: 动态更新
└── reviews: 用户评价

Analysis（分析任务）
├── competitors: 关联竞品
├── dimensions: 分析维度
├── results: 分析结果
└── report: 生成的报告

Knowledge（知识库）
├── documents: 原始文档
├── embeddings: 向量表示
└── metadata: 元数据
```
